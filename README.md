# 🗳️ SME Vote - ระบบเลือกตั้งกรรมการนักเรียน

**SME Vote** คือระบบลงคะแนนเลือกตั้งกรรมการนักเรียนอิเล็กทรอนิกส์แบบเว็บแอปพลิเคชัน (Mobile-First) ที่สามารถทำงานบน **LINE LIFF** ได้อย่างสมบูรณ์แบบ มีระบบยืนยันสิทธิ์ด้วย LINE ID ป้องกันการลงคะแนนซ้ำ (Double Voting) และมีระบบรักษาความปลอดภัยของคะแนนโหวตให้อยู่ในสถานะปิดลับแบบไม่ระบุตัวตน (Anonymous Voting) พร้อมกับแผงควบคุมผลคะแนนแบบ Real-time สำหรับแอดมิน

---

## 🚀 ฟังก์ชันหลัก (Core Features)

1. **ระบบผูกสิทธิ์นักเรียน (Student Binding):** ดึง LINE User ID ผ่าน LINE LIFF เพื่อผูกสิทธิ์เข้ากับรหัสนักเรียนที่มีสิทธิ์เลือกตั้งจริงในฐานข้อมูล ป้องกันคนนอกเข้ามาสวมสิทธิ์
2. **ระบบลงคะแนนที่ปิดลับ (Secure Anonymous Voting):**
   - ตาราง `students` จะถูกเก็บค่าธง `has_voted = TRUE` ทันทีที่โหวตเสร็จ
   - ตาราง `votes` จะเก็บเพียงตัวเลือก `candidate_id` (หรือ NULL กรณีไม่ประสงค์ลงคะแนน) โดยไม่มีการบันทึกเชื่อมโยงไปยังรหัสนักเรียนหรือ LINE ID เลย ทำให้ผู้พัฒนาหรือแอดมินไม่มีทางตรวจสอบย้อนกลับได้ว่านักเรียนคนใดลงคะแนนให้เบอร์ไหน
3. **ระบบป้องกันข้อมูลพังและการโหวตซ้ำ (Data Race Prevention):** การอัปเดตสิทธิ์และการเพิ่มบัตรลงคะแนนจะกระทำในระดับ Database Transaction ที่มีการ Lock แถวข้อมูลแบบ `FOR UPDATE` (Row Locking) หากมีคำขอการโหวตส่งเข้ามาพร้อมกันเพื่อเจาะระบบ ระบบจะยอมรับเพียงคำขอเดียวและปฏิเสธคำขอที่เหลือทันที
4. **แผงควบคุม Admin Real-time Dashboard:** แสดงกราฟคะแนนจาก Chart.js และประมวลผลสถิติจำนวนผู้มาใช้สิทธิ์ สัดส่วน และร้อยละของผู้มาลงคะแนนแบบเรียลไทม์ (Polling ทุกๆ 5 วินาที)
5. **Developer Mock Panel:** สามารถกดจำลองผู้ใช้ LINE และใส่ข้อมูลนักเรียนเพื่อความสะดวกในการรันและทดสอบระบบบนเบราว์เซอร์ปกติได้ทันที

## 📂 โครงสร้างโฟลเดอร์ของโครงการ

```
sme_vote/
├── db/
│   └── schema.sql          # ไฟล์สร้างตาราง Database ใน PostgreSQL
├── public/
│   ├── index.html          # หน้าจอหลักของนักเรียน (Voter App)
│   ├── app.js              # ลอจิกการทำงานและ LINE LIFF สำหรับฝั่งนักเรียน
│   ├── admin.html          # หน้าจอของแอดมิน (Admin Dashboard)
│   └── admin.js            # ลอจิกการดึงคะแนนและแสดงผลกราฟ Chart.js
├── server.py               # Backend API หลักพัฒนาด้วย Python FastAPI (แนะนำสำหรับระบบปัจจุบัน)
├── requirements.txt        # ไฟล์ระบุ Dependencies สำหรับฝั่ง Python
├── server.js               # Backend API ทางเลือกพัฒนาด้วย Node.js / Express
├── package.json            # ไฟล์ระบุ Dependencies สำหรับฝั่ง Node.js
└── .env                    # ไฟล์สำหรับตั้งค่าคอนฟิกูเรชันของระบบ (PORT, DB, LIFF_ID)
```

## 🛠️ ขั้นตอนการติดตั้งและเริ่มใช้งาน

### 1. การตั้งค่าฐานข้อมูล (PostgreSQL Setup)
คุณจำเป็นต้องมี PostgreSQL ทำงานอยู่ (ติดตั้งในเครื่อง หรือใช้ Cloud SQL เช่น Supabase, Render, ElephantSQL)
1. สร้างฐานข้อมูลชื่อ `sme_vote`
2. คัดลอกและสร้างตารางจากไฟล์ [db/schema.sql](file:///Users/kanapojpetchmanee/Documents/GitHub/sme_vote/db/schema.sql) หรือปล่อยให้ Backend รันอัตโนมัติเมื่อเปิดเซิร์ฟเวอร์

### 2. การตั้งค่า Environment Variables (.env)
คัดลอกไฟล์ `.env.example` ไปยัง `.env` และกรอกค่าคอนฟิกูเรชัน:
```ini
PORT=3000
DATABASE_URL=postgresql://ชื่อผู้ใช้:รหัสผ่าน@localhost:5432/sme_vote
LIFF_ID=ไอดี-LIFF-ของคุณ
```

### 3. การเปิดใช้งานเซิร์ฟเวอร์ (เลือกได้ 2 ทางเลือก)

#### ทางเลือกที่ 1: ใช้ Python FastAPI (แนะนำสำหรับการรันในสภาวะแวดล้อมปัจจุบัน)
1. ติดตั้ง Dependencies ( fastAPI, uvicorn, psycopg2-binary ฯลฯ):
   ```bash
   pip3 install -r requirements.txt
   ```
2. รันเซิร์ฟเวอร์:
   ```bash
   python3 server.py
   ```
   *ระบบจะเริ่มทำงานที่ http://localhost:3000*

#### ทางเลือกที่ 2: ใช้ Node.js / Express
1. ติดตั้ง Node Modules:
   ```bash
   npm install
   ```
2. รันเซิร์ฟเวอร์:
   ```bash
   npm start
   ```

## 🗳️ วิธีการทดสอบระบบ (Testing Guide)

1. เปิดหน้าแดชบอร์ดแอดมินที่: `http://localhost:3000/admin.html`
2. คลิกปุ่ม **"จำลองระบบและรีเซ็ตข้อมูล (Seed Data)"** ทางด้านขวาบน ระบบจะทำการสร้างฐานข้อมูลและเพิ่มข้อมูลสิทธิ์ของนักเรียนจำลอง 5 คนดังนี้:
   - รหัสนักเรียน: `65001` (กิตติ รักเรียน)
   - รหัสนักเรียน: `65002` (ศิริ งามตา)
   - รหัสนักเรียน: `65003` (วิชัย ประเสริฐยิ่ง)
   - รหัสนักเรียน: `65004` (มณี ศรีสวัสดิ์)
   - รหัสนักเรียน: `65005` (สมพร ดีงาม)
   และผู้สมัคร 3 คน (เบอร์ 1, 2, 3)
3. เปิดหน้านักเรียนเพื่อเริ่มลงคะแนนที่: `http://localhost:3000/index.html` (คุณจะเห็นกล่อง "โหมดจำลองระบบ LINE LIFF" สีเหลืองเนื่องจากไม่ได้รันใน LINE)
4. ทำการป้อน **LINE ID จำลอง** เช่น `voter_01` และกด **"ยืนยันการใช้โปรไฟล์จำลอง"**
5. ป้อนรหัสนักเรียน `65001` แล้วกดตรวจสอบสิทธิ์
6. เลือกโหวตผู้สมัครหรือกด "ไม่ประสงค์ลงคะแนน" และกดยืนยันการส่งโหวต
7. ผลคะแนนบนหน้าจอ `admin.html` จะอัปเดตแบบ Real-time ทันที!
8. ลองนำ LINE ID หรือรหัสนักเรียนเดิมไปลงคะแนนอีกรอบ ระบบจะแจ้งเตือนว่า **ใช้สิทธิ์ไปแล้ว** และสกัดกั้นการโหวตซ้ำอย่างถูกต้อง
